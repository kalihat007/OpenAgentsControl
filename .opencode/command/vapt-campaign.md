# VAPT Campaign

Create a parallel technical test campaign:

1. Define authorization and scope boundaries.
2. Map attack surface by protocol, firmware, hardware, wireless, backend, cloud, and physical access.
3. Assign parallel tracks for protocol analysis, RE, fuzzing, side-channel/fault injection, wireless, physical, network, and reporting.
4. Define evidence, severity scoring, remediation, and retest expectations.

Return test matrix, evidence plan, and approval gates.
